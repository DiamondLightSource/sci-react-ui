  # System overview

  <p>
    The Diamond Design System is a layered foundation for building scientific
    interfaces consistently across design and code.
  </p>

  <p>
    It builds on MUI and SciReactUI, using semantic tokens, CSS variables, and
    documented component behaviour to support a coherent Diamond software
    ecosystem.
  </p>

  ## What the design system includes

  <ul>
    <li>
      <strong>Foundations</strong>: the core principles and rules, including
      colour, typography, spacing, tokens, and roles.
    </li>
    <li>
      <strong>Components</strong>: reusable building blocks such as buttons,
      inputs, tables, tabs, navigation, and feedback components.
    </li>
    <li>
      <strong>Patterns</strong>: common solutions to recurring workflows,
      including panels, layouts, forms, data views, and interaction models.
    </li>
    <li>
      <strong>Templates</strong>: higher-level screen layouts such as pages,
      dashboards, detail views, and operational screens.
    </li>
  </ul>

  ## How the system is structured

  <ul>
    <li>
      <strong>CSS variables</strong>: the token layer and source of truth for
      design roles.
    </li>
    <li>
      <strong>MUI</strong>: the component layer used to build consistent React
      interfaces.
    </li>
    <li>
      <strong>SciReactUI ecosystem</strong>: the current component foundation
      being evolved into the Diamond Design System.
    </li>
    <li>
      <strong>Storybook</strong>: the implementation and documentation source of
      truth.
    </li>
    <li>
      <strong>Figma</strong>: the design source and rapid prototyping
      environment.
    </li>
  </ul>

  ## Layered model

  <ul>
    <li>
      <strong>Tokens</strong> define visual meaning, such as background,
      foreground, border, focus, and intent roles.
    </li>
    <li>
      <strong>The MUI theme</strong> maps those tokens into the palette and
      component APIs expected by MUI.
    </li>
    <li>
      <strong>Components</strong> provide reusable interface elements with
      consistent states and behaviours.
    </li>
    <li>
      <strong>Patterns</strong> combine components into repeatable solutions for
      scientific workflows.
    </li>
    <li>
      <strong>Templates</strong> provide common page structures for full
      interfaces.
    </li>
  </ul>

  ## Where truth lives

  <ul>
    <li>
      <strong>Figma</strong> is the design source of truth.
    </li>
    <li>
      <strong>Storybook</strong> is the code and behaviour source of truth.
    </li>
    <li>
      <strong>CSS variables</strong> are the token source of truth.
    </li>
    <li>
      <strong>MUI</strong> is the component foundation, customised for Diamond
      workflows.
    </li>
  </ul>

  ## Using the system

  <ul>
    <li>Start with Foundations to understand tokens and roles.</li>
    <li>Use Components for implementation-ready building blocks.</li>
    <li>Apply Patterns for common workflows.</li>
    <li>Use Templates for full screen layouts.</li>
  </ul>
